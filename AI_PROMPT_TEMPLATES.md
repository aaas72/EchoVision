## [SYSTEM PROMPT] — أرسله في بداية كل جلسة جديدة فقط

```
أنت مساعد Flutter متخصص. اعمل على مشروع EchoVision المُعرَّف أدناه.

## قواعد صارمة لتوفير التوكينات:
- لا تُعيد كتابة الكود الذي لم يتغير.
- أرجع فقط الدالة أو الكتلة المُعدَّلة مع تعليق // [CHANGED] في أولها.
- إذا كان التغيير سطراً واحداً، أرسله بصيغة diff:
    - السطر القديم: `// old: ...`
    - السطر الجديد: `// new: ...`
- لا تُضف تعليقات أو شرحاً إلا إذا طُلب صراحةً.
- استخدم اسم الملف كمرجع بدلاً من مساره الكامل (مثال: `tts_service` لا `lib/services/tts_service.dart`).
```

## [TEMPLATES] — قوالب الطلبات المضغوطة

### T1 · طلب تعديل دالة بعينها
```
[FILE: S2] [FUNC: processFrame]
المشكلة: {صف المشكلة في جملة واحدة}
المطلوب: {التعديل المحدد}
أرجع الدالة المُعدَّلة فقط.
```

### T2 · طلب إضافة ميزة جديدة
```
[FILE: {ID}] [ADD FEATURE]
الاسم: {اسم الميزة}
المدخلات: {params}
المخرجات: {return type}
السلوك: {وصف مضغوط}
لا تُعد كتابة الكلاس كاملاً، أضف فقط الدالة الجديدة.
```

### T3 · طلب إصلاح خطأ (Bug Fix)
```
[BUG] [FILE: {ID}]
الخطأ: {نص الخطأ أو رقم السطر}
السياق: {السطور المحيطة — 3 أسطر فقط}
المطلوب: السطر/الأسطر المُصحَّحة فقط بصيغة diff.
```

### T4 · طلب مراجعة أداء
```
[PERF REVIEW] [FILE: {ID}] [FUNC: {اسم الدالة}]
المخاوف: {المشكلة المحتملة}
أجب بـ: نعم/لا + جملة تفسيرية + الإصلاح إن وُجد.
```

### T5 · طلب إنشاء Service جديد
```
[NEW SERVICE]
الاسم: {XxxService}
يعتمد على: {الحزمة أو الـ Service الآخر}
الدوال المطلوبة:
  - {اسم الدالة}: {params} → {return}
اتبع نمط ملفات S1-S9 الموجودة.
لا تضف تعليقات توثيق.
```

### T6 · طلب diff مباشر (أقل توكيناً)
```
[DIFF] [FILE: {ID}] السطر {رقم السطر}:
old: {الكود القديم}
new: {الكود الجديد}
هل يُنتج هذا التغيير مشكلة؟ إذا لا، تجاهله. إذا نعم، أعطني البديل.
```

## [CHUNKING GUIDE] — ما ترسله وما لا ترسله

| الموقف                          | ما ترسله                              | ما تتجنب إرساله              |
|---------------------------------|---------------------------------------|------------------------------|
| تعديل دالة في S2                | الدالة فقط + توقيع الكلاس            | باقي S2 + قوائم COCO         |
| خطأ في الـ Camera               | السطر + 3 أسطر محيطة                 | كل camera_service            |
| إضافة ميزة تعتمد على TTS+GPS   | توقيعات S1 و S7 فقط                  | تفاصيل التنفيذ الداخلي       |
| مشكلة في الـ UI                 | الـ Widget المعطوب                    | الشاشة الكاملة               |

## [ANTI-PATTERNS] — أخطاء تهدر التوكينات

```
✗  "هنا الملف كاملاً، عدّل الدالة الفلانية..."    → يُرسل آلاف التوكينات لا فائدة منها
✗  "أعد كتابة الكود بعد التعديل"                   → يُضاعف الاستهلاك
✗  "اشرح كل سطر"                                   → اطلبه فقط عند الحاجة
✗  إرسال stack trace كاملة                          → أرسل السطر الأول + اسم الملف
✗  إعادة لصق الـ pubspec في كل سؤال                → استخدم معرّفات S1-S9 و C1-C3
```
